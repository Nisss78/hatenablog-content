<h3>第33章：時間差の魔術</h3>

<p>「非同期の試練」へと続く通路は、これまでとは異なる雰囲気を漂わせていた。壁の古代文字が一定のリズムで点滅し、まるで呼吸しているかのように見えた。</p>

<p>「なんだろう、この感覚」エリナが通路の壁に手を触れた。「壁から空気が漏れてるみたい。何かが動いている」</p>

<p>リアムも剣の柄に手をかけた。「気配が途切れ途切れだ。敵がいるのか、いないのか、判別できない」</p>

<p>カイトは通路の天井を見上げた。そこには、複雑な図形が時折光っては消え、光っては消えを繰り返していた。</p>

<p>「非同期処理だ」カイトは呟いた。「複数の処理が、時間差で実行されている。ある処理が終わるのを待たずに、次の処理が始まる。この通路自体が、その仕組みを表現している」</p>

<p>「時間差で実行？」セシリアが首をかしげた。「魔法なら、詠唱が終わってから効果が出るのが普通ですわ」</p>

<p>「それが同期的な処理だ」カイトは説明した。「一つずつ順番にこなしていく。でも、非同期は違う。例えば、レストランで注文した料理が来るのを待っている間、他の客の注文も受け付けるだろ？ それと同じだ。一つの処理が終わるのを待たずに、他の処理も同時に進める」</p>

<p>「効率的ではあるわね」エリナが頷いた。「でも、複雑そう」</p>

<p>「複雑さはある」カイトは認めた。「どの処理がいつ終わるか、管理しなきゃいけないからな。でも、その分、柔軟な対応ができる」</p>

<p>通路の突き当たりに、半透明の扉が現れた。扉の向こうが見えるが、実体としての扉は存在しなかった。</p>

<p>カイトが手を伸ばすと、指が扉をすり抜けた。</p>

<p>「幻影か？」リアムが剣を振った。剣も扉を通り抜けた。</p>

<p>「違う」カイトは首を振った。「これはasyncの扉だ。すぐに開くんじゃなくて、『準備ができたら』開く。今はまだ準備中だ」</p>

<p>「いつ準備ができるの？」セシリアが尋ねた。</p>

<p>「それが非同期の面白いところだ」カイトは扉を観察した。「正確な時間はわからない。でも、何らかの条件が満たされたら、自動的に開くはずだ」</p>

<p>四人は扉の前で待った。</p>

<p>一分。二分。</p>

<p>「これ、本当に開くのか？」リアムが不審そうに言った。「何も起こっていないように見えるが」</p>

<p>「非同期処理は見えないところで動いていることが多い」カイトは説明した。「表面上は何も変わらないように見えても、裏では準備が進んでいる」</p>

<p>その時、扉の向こうから光が漏れた。</p>

<p>カイトが目を凝らすと、光の中に文字が浮かんでいた。</p>

<pre><code>async function openDoor() {
    await prepareRoom();
    await summonGuardian();
    return door.open();
}</code></pre>

<p>「コードだ」カイトは言った。「この扉は、async関数で制御されている。部屋を準備して、守護者を召喚して、それから開く。全部順番に実行されているが、俺たちには準備中のことが見えていない」</p>

<p>「つまり、今はprepareRoomが実行されている？」エリナが尋ねた。</p>

<p>「いや、もう次の段階に進んでいるかもしれない。awaitは、一つの処理が終わるのを待つ構文だ。prepareRoomが終わったらsummonGuardian、それが終わったらdoor.open。処理が終わるのを待ちながら、順番に実行されている」</p>

<p>扉の光が強くなった。</p>

<p>そして、唐突に、扉が開いた。</p>

<p>「開いた！」セシリアが叫んだ。</p>

<p>四人は扉をくぐった。</p>

<p>その部屋は、カイトがこれまで見た中で最も奇妙な空間だった。</p>

<p>部屋の中に、複数の時間が同時に存在していた。</p>

<p>左側では、石のブロックが空中に浮かんでいた。右側では、同じブロックが床に落ちて砕けていた。正面では、ブロックがまだ形を成していない砂の状態で存在していた。</p>

<p>「過去と現在と未来が、同時にある？」エリナが戸惑った声を上げた。</p>

<p>「違う」カイトは首を振った。「非同期だ。複数の処理が、それぞれのタイミングで実行されている。左のブロックはまだ浮遊中、右のブロックは落下完了、正面のブロックは生成待ち。三つの処理が並行して走っているんだ」</p>

<p>部屋の中央に、奇妙な生き物がいた。</p>

<p>体は半透明で、中に複数の光が浮遊していた。光はそれぞれ異なる色をしていて、ある光は速く動き、ある光はゆっくり動いていた。</p>

<p>「asyncの守護者だ」カイトは直感した。「内部で複数の処理を並行して実行している」</p>

<p>守護者がカイトたちに気づいた。</p>

<p>声が響いた——いや、複数の声が、時間差で響いた。</p>

<p>「訪問者よ」「訪問者よ」「訪問者よ」</p>

<p>三つの声が、ずれて聞こえた。まるでエコーのように。</p>

<p>「我は」「我は」「我は」</p>

<p>「非同期の守護者」「守護者」「守護者」</p>

<p>「なんだこれは」リアムは剣を構えた。「攻撃してくるのか？」</p>

<p>「いや、これは挨拶だ」カイトは分析した。「一つの言葉を発しているのに、時間差で届いている。asyncの性質だ」</p>

<p>守護者の中の光が動いた。赤い光が前に出て、カイトに向かって飛んできた。</p>

<p>カイトは避けた——と同時に、青い光が別の角度から飛んできた。</p>

<p>「二つ同時に！」エリナが魔法で防御した。</p>

<p>「同時じゃない」カイトは訂正した。「赤い光が飛んでいる間に、青い光も飛び始めた。処理が並行して実行されているんだ」</p>

<p>四つの光が、四方から同時に——いや、わずかに時間差で——飛んできた。</p>

<p>四人は散開した。</p>

<p>「どうやって戦うの？」セシリアが叫びながら魔法の盾を展開した。「攻撃がいつ来るかわからないじゃない！」</p>

<p>カイトは思考を巡らせた。非同期処理に対抗するには、どうすればいいか。</p>

<p>答えは一つだった。</p>

<p>「awaitだ」カイトは叫んだ。「攻撃を待つんじゃなくて、攻撃が来るタイミングを予測して、その瞬間に対処する。非同期処理を同期的に扱う」</p>

<p>「どういうこと？」リアムが剣で光を払いながら尋ねた。</p>

<p>「awaitは、非同期処理の完了を待つ構文だ」カイトは説明した。「この守護者の中で、光がいつ飛ぶかを管理している関数があるはずだ。その関数にawaitを使えば、光が飛ぶタイミングを把握できる」</p>

<p>「でも、どうやって？」エリナが尋ねた。</p>

<p>カイトは守護者を見つめた。半透明の体の中で、光が渦巻いている。</p>

<p>「コードを書くんだ」カイトは空中に指を走らせた。「この守護者の非同期処理を、俺たちのやり方で制御する」</p>

<p>光の文字が空中に浮かんだ。</p>

<pre><code>async function predictAttack(guardian) {
    const redLight = await guardian.launchRedLight();
    const blueLight = await guardian.launchBlueLight();
    
    return {
        first: redLight.timing,
        second: blueLight.timing
    };
}</code></pre>

<p>「予測関数だ」カイトは言った。「守護者の攻撃タイミングを、awaitで待ってから返す。これで、いつ攻撃が来るかわかる」</p>

<p>コードが守護者に吸い込まれた。</p>

<p>カイトの脳内に、情報が流れ込んできた。</p>

<p>『赤い光：3秒後に発射』</p>

<p>『青い光：1.5秒後に発射』</p>

<p>『緑の光：4秒後に発射』</p>

<p>『黄の光：2秒後に発射』</p>

<p>「わかった！」カイトは叫んだ。「次は青、その次が黄、それから赤、最後が緑だ！」</p>

<p>「何秒？」エリナが尋ねた。</p>

<p>「青は今から1.5秒後、黄は2秒後。ほぼ同時だ」</p>

<p>四人は構えた。</p>

<p>1.5秒後。青い光が飛んできた。カイトは右にステップした。光が空を切った。</p>

<p>2秒後。黄色い光が来た。セシリアが盾で弾いた。</p>

<p>「当たった！」セシリアが喜んだ。</p>

<p>「まだ終わらない」カイトは警告した。「赤と緑が残っている」</p>

<p>3秒後。赤い光。リアムが剣で切り裂いた。</p>

<p>4秒後。緑の光。エリナが魔法で相殺した。</p>

<p>「回避成功」カイトは息を吐いた。「でも、これだけじゃ勝てない。予測できても、攻撃を防ぐだけで精一杯だ」</p>

<p>守護者の中の光が、再び動き始めた。</p>

<p>「次のラウンドだ」カイトは言った。「もっと攻撃が激しくなるはずだ。非同期処理は、並行して実行できる数を増やせるからな」</p>

<p>守護者の体の中で、光の数が増えた。六つ、八つ、十。</p>

<p>「十個の攻撃を予測するのは無理だ」エリナが顔をしかめた。</p>

<p>「予測するんじゃない」カイトは首を振った。「別の方法を考える。非同期処理には、もう一つ重要な概念がある」</p>

<p>「何？」リアムが尋ねた。</p>

<p>「Promiseだ」カイトは答えた。「処理の完了を表すオブジェクト。成功か失敗か、最終的な結果を約束する」</p>

<p>カイトは再び空中にコードを書いた。</p>

<pre><code>function defeatGuardian(guardian) {
    return new Promise((resolve, reject) => {
        guardian.attacks.forEach(attack => {
            attack.cancel();
        });
        
        if (guardian.isDefenseless()) {
            resolve("守護者を無力化");
        } else {
            reject("無力化失敗");
        }
    });
}

async function battle() {
    try {
        const result = await defeatGuardian(guardian);
        console.log(result);
    } catch (error) {
        console.error(error);
    }
}</code></pre>

<p>「Promiseを作る」カイトは説明した。「守護者の全攻撃をキャンセルして、無力化できたら成功、できなかったら失敗。結果を約束するオブジェクトだ」</p>

<p>「それをどう使うの？」セシリアが尋ねた。</p>

<p>「awaitで待つ」カイトは言った。「Promiseが解決するまで、次の処理に進まない。非同期処理を、同期的に扱える」</p>

<p>コードが守護者に向かって飛んだ。</p>

<p>守護者の体の中で、光が不規則に動き始めた。十個の光が、それぞれ異なる方向に動いている。</p>

<p>「Promiseが実行されている」カイトは解説した。「各攻撃をキャンセルしようとしている」</p>

<p>光が次々と消えていった。</p>

<p>一つ、二つ、三つ……</p>

<p>しかし、最後の三つが消えなかった。むしろ、より激しく動き始めた。</p>

<p>「rejectされた」カイトは顔をしかめた。「無力化失敗。この守護者は、最後の三つの攻撃をキャンセルできない仕組みになっている」</p>

<p>残った三つの光——紫、白、黒——が、カイトたちを包囲した。</p>

<p>「どうする？」リアムが剣を構えた。「三つなら対処できるかもしれないが」</p>

<p>「いや、こいつらは特別だ」カイトは警戒した。「キャンセルできないということは、システムの核に近い存在だ。async/awaitの根本に関わる何かかもしれない」</p>

<p>三つの光が、同時に——本当に同時に——飛んできた。</p>

<p>awaitで待っても、予測しても、防げなかった。</p>

<p>カイトは咄嗟に判断した。</p>

<p>「全員、地面に伏せろ！」</p>

<p>四人が地面に倒れた瞬間、三つの光が頭上を通過した。光は壁にぶつかり、爆発した。</p>

<p>「くそっ、危なかった」カイトは起き上がった。「今の攻撃、awaitじゃ追いつけない」</p>

<p>「どういうこと？」エリナが尋ねた。</p>

<p>「三つの光が、真に並行して実行されたんだ」カイトは説明した。「時間差じゃなくて、完全に同時。Promise.allの状態だ」</p>

<p>「Promise.all？」セシリアが首をかしげた。</p>

<p>「複数のPromiseを同時に実行して、全部終わるのを待つ構文だ」カイトは空中に書いた。</p>

<pre><code>const [purple, white, black] = await Promise.all([
    guardian.launchPurpleLight(),
    guardian.launchWhiteLight(),
    guardian.launchBlackLight()
]);</code></pre>

<p>「これだと、三つの処理が同時に実行されて、全部終わるまで待つ。個別にawaitするより速い」</p>

<p>「じゃあ、どうやって防ぐの？」リアムが尋ねた。</p>

<p>カイトは考えた。</p>

<p>「防ぐんじゃなくて、先に処理を終わらせる」</p>

<p>「どうやって？」</p>

<p>「Promise.raceだ」カイトは答えた。「複数のPromiseのうち、最初に終わったものの結果を採用する。先に俺たちの処理を終わらせれば、守護者の攻撃より速く結果を出せる」</p>

<p>カイトは三度、コードを書いた。</p>

<pre><code>const result = await Promise.race([
    guardian.finishAttack(),
    ourTeam.launchCounterMagic()
]);

if (result === "counterSuccess") {
    guardian.surrender();
}</code></pre>

<p>「競争だ」カイトは言った。「守護者の攻撃が終わるのと、俺たちの反撃が届くの、どっちが速いか」</p>

<p>「エリナ、リアム、セシリア、全員の力を集めろ。一つの魔法にするんだ。俺がコードで誘導する」</p>

<p>三人が頷いた。</p>

<p>エリナが炎の魔法を、リアムが聖なる光を、セシリアが風の魔法を、それぞれ放った。</p>

<p>三つの魔法が空中で交差した。</p>

<p>カイトはそれをコードで包み込んだ。</p>

<pre><code>async function counterMagic() {
    const fire = await elina.fireMagic();
    const light = await liam.holyLight();
    const wind = await cecilia.windMagic();
    
    return combine(fire, light, wind);
}</code></pre>

<p>「組み合わせた」カイトは叫んだ。「次は、守護者の攻撃と競争させる」</p>

<p>カイトはコードを守護者に向かって放った。</p>

<p>三つの光——紫、白、黒——が再び飛んできた。</p>

<p>しかし、カイトたちの反撃の方が、一瞬早かった。</p>

<p>合成魔法が守護者に命中した。</p>

<p>守護者が叫んだ——初めて、単一の声で。</p>

<p>「敗北を認める」</p>

<p>体の中の光が、全て消えた。半透明の体が、徐々に実体を帯びていった。</p>

<p>やがて、一人の老人が立っていた。古代の魔法使いの姿だ。</p>

<p>「私は、非同期の守護者だった者だ」老人は言った。「君たちは、async/awaitの本質を理解した。処理を並行して実行し、必要な時には待ち、競争させる。時間を味方につける術を会得した」</p>

<p>カイトは息を整えた。「非同期処理は、確かに強力だ。一つずつこなすより、ずっと効率的になる」</p>

<p>「だが、注意せよ」老人は警告した。「非同期処理は複雑になりやすい。どの処理がいつ終わるか、管理を怠ると、予期せぬバグを生む。常に、処理の流れを把握し続けるのだ」</p>

<p>老人は手を振った。</p>

<p>部屋の奥に、新たな扉が現れた。</p>

<p>「次は、『コールバックの試練』だ。非同期処理の、さらに根源にある概念だ。備えよ」</p>

<p>老人の姿が消えた。</p>

<p>「コールバックか」カイトは呟いた。「関数を引数として渡して、処理が終わった時に呼び出してもらう仕組みだ。非同期処理の基礎だな」</p>

<p>「どんどん難しくなっていくわね」エリナがため息をついた。</p>

<p>「でも、面白い」カイトは笑った。「この世界の魔法は、プログラミングの概念を忠実に再現している。学ぶたびに、新しい発見がある」</p>

<p>リアムがカイトの肩を叩いた。「頼りにしているよ、カイト。君の知識がなければ、ここまで来れなかった」</p>

<p>セシリアも頷いた。「私も、カイトさんの世界の技術に興味を持ち始めましたわ。もし元の世界に戻れたら、ぜひ教えてください」</p>

<p>「それは約束するよ」カイトは答えた。「まずは、この試練を全部クリアして、地上に戻ろう」</p>

<p>四人は、新たな扉へと歩き出した。</p>

<p>カイトの心には、確信があった。</p>

<p>（この実験場の最深部には、何か重要なものがある。古代の魔法使いたちが、プログラミングの概念を使って何をしようとしたのか。その答えが、きっとあるはずだ）</p>

<p>地下の迷宮は、まだ終わらない。</p>

<hr>

<p><strong>【プログラミング解説】</strong></p>

<ul>
<li><strong>async/await</strong>：非同期処理を同期的に書ける構文。awaitで処理の完了を待てる。</li>
<li><strong>Promise</strong>：非同期処理の最終的な結果を表すオブジェクト。成功(resolve)か失敗(reject)かを約束する。</li>
<li><strong>Promise.all</strong>：複数のPromiseを並行して実行し、全て完了するのを待つ。</li>
<li><strong>Promise.race</strong>：複数のPromiseのうち、最初に完了したものの結果を採用する。</li>
</ul>
