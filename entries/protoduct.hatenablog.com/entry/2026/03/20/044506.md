<h3>第35章：コールバックの深淵</h3>

<p>共鳴の余韻がまだ体内に残る中、四人は「コールバックの試練」への扉をくぐった。</p>

<p>カイトの手のひらには、前章の戦いで刻まれた複雑な魔法陣が薄く光を放っていた。エリナ、リアム、セシリアの三人もまた、それぞれの体に微かな光の紋様が浮かんでいるのを感じていた。</p>

<p>「共鳴って、何だったんだろう」エリナが自分の手を見つめながら呟いた。「私たち四人の力が、一つになったような感覚」</p>

<p>「同期的な結合だったんだと思う」カイトは答えた。「四人が別々に動くんじゃなくて、全員が同じタイミングで力を合わせる。非同期処理を、一時的に同期処理に変えたんだ」</p>

<p>「それができたから、守護者に勝てたのね」セシリアが頷いた。</p>

<p>通路は緩やかに下っていた。壁の古代文字は、これまで見たどの試練よりも複雑に絡み合っていた。</p>

<p>「文字が動いている」リアムが指摘した。「見てくれ。あの文字、さっきは上にあったのに、今は下にある」</p>

<p>カイトが壁を観察した。確かに、古代文字が一定の法則に従って移動していた。</p>

<p>「入れ替わっているんじゃない」カイトは分析した。「呼び出されているんだ。ある文字が、別の文字を呼び出している」</p>

<p>「呼び出す？」エリナが首をかしげた。</p>

<p>「コールバックだ」カイトは答えた。「処理が終わった時に、次の処理を呼び出す仕組み。この通路自体が、コールバックの概念で作られている」</p>

<p>通路の突き当たりに、巨大な鏡が現れた。</p>

<p>鏡の中には、四人の姿が映っていた——しかし、鏡像は四人の動きとは異なる動きをしていた。</p>

<p>鏡の中のカイトが、剣を振った。現実のカイトは剣を持っていない。</p>

<p>「幻覚か？」リアムが警戒した。</p>

<p>「違う」カイトは鏡に近づいた。「これは、未来を映しているんじゃない。過去のコールバックを映している」</p>

<p>「過去？」セシリアが尋ねた。</p>

<p>「コールバックは、ある処理が終わった時に呼び出される関数だ」カイトは説明した。「この鏡は、過去の私たちが登録したコールバックを、今実行している。つまり、鏡の中の姿は、過去に私たちが『終わったらやるべきこと』として残した行動だ」</p>

<p>鏡の中の四人が、同時にこちらを向いた。</p>

<p>「来い」鏡の中のカイトが言った。「約束された場所へ」</p>

<p>鏡の表面が波打った。</p>

<p>「入るのか？」エリナが戸惑った声を上げた。</p>

<p>「入るしかない」カイトは決断した。「コールバックは、登録されたら必ず実行される。逃避はできない」</p>

<p>四人は鏡の中へと歩いていった。</p>

<p>鏡の向こう側は、奇妙な空間だった。</p>

<p>無数の部屋が、蜘蛛の巣のように繋がっていた。各部屋には、異なる時代の四人がいた。</p>

<p>左の部屋には、試練を始めたばかりの四人がいる。</p>

<p>右の部屋には、傷だらけで戦っている四人がいる。</p>

<p>上の部屋には、平和そうに休んでいる四人がいる。</p>

<p>「時間が分岐している？」リアムが驚いた。</p>

<p>「違う」カイトは首を振った。「コールバックのチェーンだ。一つの処理が終わるたびに、次の処理が呼び出される。その繰り返しが、ここでは視覚化されている」</p>

<p>空間の中央に、巨大な球体が浮いていた。</p>

<p>球体の中には、無数の光の糸が絡み合っていた。糸は各部屋へと伸びていて、部屋同士を繋いでいる。</p>

<p>「これがコールバックの守護者か」カイトは推測した。</p>

<p>球体が振動した。</p>

<p>声が響いた——無数の声が、次々と。</p>

<p>「第一の処理」「完了」「第二の処理」「呼び出し」「第三の処理」「待機」「第四の処理」「実行中」</p>

<p>「うるさい」エリナが耳を塞いだ。「何が何だかわからない」</p>

<p>「コールバック地獄だ」カイトは顔をしかめた。「コールバックの中にコールバック、その中にまたコールバック……。処理が深くネストして、何が起きているかわからなくなる。プログラミングでよくある問題だ」</p>

<p>「どうやって解決するの？」セシリアが尋ねた。</p>

<p>「コールバックを整理する」カイトは答えた。「処理の流れを可視化して、依存関係を明確にする」</p>

<p>カイトは空中に指を走らせた。</p>

<pre><code>// コールバック地獄の例
firstFunction(function(result1) {
    secondFunction(result1, function(result2) {
        thirdFunction(result2, function(result3) {
            fourthFunction(result3, function(result4) {
                // どこまで深くなるんだ…
            });
        });
    });
});</code></pre>

<p>「これが今の状態だ」カイトは説明した。「処理がどんどん深くなって、全体像が見えない」</p>

<p>「じゃあ、どうすればいいの？」リアムが尋ねた。</p>

<p>「名前付き関数に分ける」カイトは答えた。「無名関数のままネストするから、複雑になるんだ」</p>

<p>カイトは新たなコードを書いた。</p>

<pre><code>// 整理されたコールバック
function handleFirst(result1) {
    secondFunction(result1, handleSecond);
}

function handleSecond(result2) {
    thirdFunction(result2, handleThird);
}

function handleThird(result3) {
    fourthFunction(result3, handleFourth);
}

function handleFourth(result4) {
    // 完了
}

firstFunction(handleFirst);</code></pre>

<p>「各処理に名前を付けて、別々に定義する」カイトは言った。「これで、処理の流れが追いやすくなる」</p>

<p>コードが球体に吸い込まれた。</p>

<p>球体の中の光の糸が、徐々にほどけていった。複雑に絡み合っていた糸が、一本の直線に近い形に整列していく。</p>

<p>「効いている」エリナが希望を持った。</p>

<p>しかし、球体の奥から、新たな糸が現れた。</p>

<p>「まだある」カイトは警戒した。「コールバック地獄は、表層だけ整理してもダメだ。根本的な構造を変えないと」</p>

<p>新たな糸は、球体の奥底から湧き出ていた。まるで、無限に増殖しているかのように。</p>

<p>「どうするの？」セシリアが不安そうに尋ねた。</p>

<p>カイトは考えた。</p>

<p>「async/awaitに変換する」</p>

<p>「え？」エリナが戸惑った。</p>

<p>「コールバック方式から、Promiseベースの方式への移行だ」カイトは説明した。「コールバックの複雑さを解消するために、Promiseが生まれた。そして、Promiseをさらに使いやすくするために、async/awaitが生まれた。技術の進化の流れだ」</p>

<p>カイトは、これまでで最も複雑なコードを書き始めた。</p>

<pre><code>// コールバックをPromiseで包む
function promisify(callbackFunction) {
    return function(...args) {
        return new Promise((resolve, reject) => {
            callbackFunction(...args, (error, result) => {
                if (error) reject(error);
                else resolve(result);
            });
        });
    };
}

// async/awaitで書き直す
async function organizedProcess() {
    try {
        const result1 = await promisify(firstFunction)();
        const result2 = await promisify(secondFunction)(result1);
        const result3 = await promisify(thirdFunction)(result2);
        const result4 = await promisify(fourthFunction)(result3);
        return result4;
    } catch (error) {
        console.error("処理中にエラーが発生:", error);
    }
}</code></pre>

<p>「コールバックをPromiseで包んで、async/awaitで同期的に書く」カイトは言った。「これで、複雑なネストが、シンプルな直列処理になる」</p>

<p>コードが球体に向かって飛んだ。</p>

<p>球体が激しく振動した。</p>

<p>中の光の糸が、一瞬で全てほどけた。そして、一本の太い光の柱へと変化した。</p>

<p>「成功か？」リアムが尋ねた。</p>

<p>「まだだ」カイトは油断しなかった。「柱の中に、何かいる」</p>

<p>光の柱の中から、一人の人物が現れた。</p>

<p>それは、カイトだった。</p>

<p>「え？」カイトは戸惑った。「俺？」</p>

<p>「私は、コールバックの守護者だ」もう一人のカイトが言った。「君の過去の記憶、君の経験、君の判断パターン……それら全てが、コールバックとして登録されていた。君がここに来た時から、ずっと準備されていたんだ」</p>

<p>「俺自身がコールバック？」カイトは驚いた。</p>

<p>「正確には、君の『過去の決断』がコールバックとして登録されていた」守護者は説明した。「君はこれまで、数々の選択をしてきた。その都度、『後でどうするか』を無意識に登録していた。そのコールバックが、今、一斉に実行されたんだ」</p>

<p>「じゃあ、この試練は……」</p>

<p>「自分自身との対話だ」守護者は頷いた。「過去の自分が残した課題に、向き合うことだ」</p>

<p>守護者が手を伸ばした。</p>

<p>「来い。君の過去のコールバックを、一つずつ処理しよう」</p>

<p>カイトは仲間たちを振り返った。</p>

<p>「一人で行くべきか」カイトは言った。「これは、俺自身の問題だ」</p>

<p>「馬鹿言わないで」エリナがカイトの手を握った。「私たちはチームでしょ。一人で抱え込まないで」</p>

<p>リアムも頷いた。「共鳴の試練で、俺たちは一つになったんだ。今さら離れる必要はない」</p>

<p>セシリアも微笑んだ。「私たちも、カイトさんの過去に関わらせてください。一緒に解決しましょう」</p>

<p>カイトは感謝の気持ちで胸がいっぱいになった。</p>

<p>「ありがとう」カイトは言った。「一緒に行こう」</p>

<p>四人は、守護者の導きで光の柱の中へと入っていった。</p>

<p>中は、カイトの記憶の空間だった。</p>

<p>子供の頃のカイトが、初めてプログラミングに触れている。</p>

<p>高校生のカイトが、複雑なコードに苦戦している。</p>

<p>社会人のカイトが、大きなプロジェクトを任されている。</p>

<p>そして、異世界に転移した瞬間のカイトがいる。</p>

<p>「これが……俺のコールバック」カイトは呟いた。</p>

<p>「君は、プログラミングを学び始めた時から、『もっと効率的に』『もっと美しく』を追求してきた」守護者は言った。「その過程で、解決できなかった問題、見過ごしたバグ、諦めた最適化……それらがコールバックとして蓄積されていた」</p>

<p>「じゃあ、この試練は、それらを解決しろってこと？」カイトは尋ねた。</p>

<p>「違う」守護者は首を振った。「解決する必要はない。ただ、認めることだ。完璧じゃなくても、君は前に進んできた。それを認めることだ」</p>

<p>カイトは、自分の記憶を見つめた。</p>

<p>失敗もあった。挫折もあった。でも、その度に、前に進んできた。</p>

<p>「認める」カイトは言った。「俺は完璧じゃない。でも、俺は進み続ける。それが、俺のプログラミング哲学だ」</p>

<p>守護者が微笑んだ。</p>

<p>「その答えでいい」</p>

<p>守護者の姿が、光の中に溶けていった。</p>

<p>「君は、コールバックの試練をクリアした。過去の自分と向き合い、受け入れる勇気を持った。それが、真のプログラマーの資質だ」</p>

<p>空間が揺らぎ、元の通路に戻った。</p>

<p>四人は、新たな扉の前に立っていた。</p>

<p>「次は何？」エリナが尋ねた。</p>

<p>「『イベントループの深淵』だ」カイトは扉に刻まれた文字を読んだ。「非同期処理の、心臓部だ」</p>

<p>「まだ続くのね」セシリアがため息をついた。</p>

<p>「でも、終わりに近づいているはずだ」カイトは言った。「実験場の構造を考えると、核心部分に近い概念が出てきている」</p>

<p>リアムがカイトの肩を叩いた。「休息が必要なら言ってくれ。君が無理すると、俺たちも困る」</p>

<p>「大丈夫」カイトは笑った。「むしろ、調子が出てきたよ。この世界の魔法システム、本当に興味深い」</p>

<p>四人は、イベントループへの扉を開いた。</p>

<p>地下の迷宮は、さらに深く続いていた。</p>

<p>しかし、カイトの心には迷いはなかった。</p>

<p>（プログラミングの概念が、魔法として具現化されている。これは、単なる偶然じゃない。古代の魔法使いたちは、何か大きな目的のために、この実験場を作ったんだ。その目的を知るために、俺は進み続ける）</p>

<p><strong>【プログラミング解説】</strong></p>

<ul>
<li><strong>コールバック関数</strong>：処理が終わった時に呼び出される関数。非同期処理の基本的な仕組み。</li>
<li><strong>コールバック地獄</strong>：コールバックの中にコールバックを入れ子にしすぎて、コードが読みづらくなる現象。</li>
<li><strong>Promise化（promisify）</strong>：コールバック方式の関数をPromise方式に変換すること。</li>
<li><strong>async/await</strong>：Promiseをより同期的に書ける構文。コールバック地獄の解決策の一つ。</li>
</ul>
