<h3>第36章：イベントループの深淵</h3>

<p>扉の向こう側は、これまでとは全く異なる空間だった。</p>

<p>天井も壁も床もない。ただ、無数の歯車が空中に浮かんでいた。巨大な歯車、小さな歯車、中くらいの歯車——それらが絶えず回転し、互いに噛み合いながら、複雑なリズムを刻んでいた。</p>

<p>「これは……」カイトは圧倒された。「機械仕掛けの世界？」</p>

<p>「違う」エリナが首を振った。「見て。歯車の表面に、文字が浮かんでいる」</p>

<p>カイトが近くの歯車を観察した。確かに、各歯車の表面には、古代文字と魔法陣が刻まれていた。そして、文字は流動的に変化していた。</p>

<p>「関数だ」カイトは気づいた。「各歯車が、一つの関数を表している」</p>

<p>「関数？」リアムが眉をひそめた。</p>

<p>「処理の単位」カイトは説明した。「プログラムの中で、一つの仕事をするコードの塊。この空間全体が、巨大な処理系を表している」</p>

<p>セシリアが空を見上げた。「あそこ、何かが浮かんでいるわ」</p>

<p>空間の中央に、巨大な円環が浮かんでいた。円環は、時計回りにゆっくりと回転していた。円環の周りには、無数の光の玉が浮かんでいて、円環に向かって流れていたり、円環から遠ざかっていたりしていた。</p>

<p>「あれがイベントループだ」カイトは確信した。「非同期処理の心臓部」</p>

<p>「心臓部？」エリナが尋ねた。</p>

<p>「JavaScriptのようなシングルスレッドの処理系では、一度に一つの処理しかできない」カイトは説明した。「でも、非同期処理が必要な場面は多い。タイマー、ネットワーク通信、ユーザー入力……それらを管理するのが、イベントループだ」</p>

<p>カイトは空中に指を走らせた。</p>

<pre><code>// イベントループの概念
while (eventQueue.waitForEvent()) {
    const event = eventQueue.getNextEvent();
    event.execute();
}</code></pre>

<p>「イベントループは、イベントを待ち受けて、イベントが来たら実行する。この単純なループが、全ての非同期処理を支えている」</p>

<p>「単純？」リアムが疑わしげに言った。「これだけ複雑な空間が、単純な仕組みで動いているのか？」</p>

<p>「概念自体は単純だ」カイトは答えた。「でも、実装は複雑になる。特に、複数の種類のタスクが混在するときは」</p>

<p>四人は、イベントループの円環に近づいていった。</p>

<p>円環の周りには、三種類の領域があった。</p>

<p>一つ目の領域には、「コールスタック」と書かれた碑があった。その周りには、積み木のように重なった光のブロックがあった。</p>

<p>二つ目の領域には、「タスクキュー」と書かれた碑があった。その周りには、行列を作って待っている光の玉があった。</p>

<p>三つ目の領域には、「マイクロタスクキュー」と書かれた碑があった。その周りにも、行列があった——ただし、タスクキューよりも小さな光の玉が並んでいた。</p>

<p>「三つの領域」カイトは分析した。「コールスタックが現在実行中の処理。タスクキューが通常の非同期タスク。マイクロタスクキューが優先度の高いタスク」</p>

<p>「優先度？」セシリアが尋ねた。</p>

<p>「マイクロタスクは、通常のタスクよりも先に実行される」カイトは説明した。「Promiseの解決などが、マイクロタスクとして扱われる」</p>

<p>「じゃあ、マイクロタスクが多いと、通常のタスクがいつまで経っても実行されないってこと？」エリナが心配そうに言った。</p>

<p>「その通り」カイトは頷いた。「マイクロタスクが無限に発生すると、イベントループがブロックされる。飢餓状態って呼ばれる現象だ」</p>

<p>その時、円環が激しく振動した。</p>

<p>「誰かが来る」リアムが剣に手をかけた。</p>

<p>円環の中から、一人の女性が現れた。</p>

<p>長い銀髪、冷徹な表情、そして、体全体が半透明だった。彼女の体の中には、無数の光の流れが見えた——まるで、彼女自身が処理の流れそのものであるかのように。</p>

<p>「ようこそ、異世界のプログラマー」女性が言った。「私は、イベントループの守護者。名前は、ループ」</p>

<p>「ループ？」カイトは戸惑った。</p>

<p>「イベントループそのものだから」ループは淡々と答えた。「この実験場の全ての非同期処理は、私を通じて実行される」</p>

<p>「何のために？」セシリアが尋ねた。</p>

<p>「古代の魔法使いたちは、魔法を効率的に実行する方法を探していた」ループは説明した。「一度に一つの魔法しか唱えられない制約の中で、複数の処理をこなすために、イベントループの概念を導入した」</p>

<p>「それで、この巨大な仕組みを？」カイトは感心した。</p>

<p>「魔法と技術の融合」ループは頷いた。「しかし、イベントループには弱点がある。一つの処理が長すぎると、他の処理が待たされる」</p>

<p>ループが手をかざした。</p>

<p>突然、巨大な光のブロックが、コールスタックの上に現れた。</p>

<p>「これは、重い処理だ」ループは言った。「この処理が終わるまで、他の処理は実行されない」</p>

<p>タスクキューの光の玉が、不安げに揺れた。マイクロタスクキューの光の玉も同様だった。</p>

<p>「どうすればいい？」エリナがカイトに尋ねた。</p>

<p>「処理を分割する」カイトは答えた。「長い処理を、短い処理の連続に分解する」</p>

<p>カイトは空中に指を走らせた。</p>

<pre><code>// 重い処理（悪い例）
function heavyProcess() {
    for (let i = 0; i < 1000000000; i++) {
        // 長い計算
    }
}

// 分割された処理（良い例）
function chunkedProcess(start, end, callback) {
    const chunkSize = 10000;
    let i = start;
    
    function processChunk() {
        const chunkEnd = Math.min(i + chunkSize, end);
        while (i < chunkEnd) {
            // 計算
            i++;
        }
        
        if (i < end) {
            setTimeout(processChunk, 0);
        } else {
            callback();
        }
    }
    
    processChunk();
}</code></pre>

<p>「setTimeoutで、次の処理をキューに戻す」カイトは説明した。「これで、イベントループが他の処理を実行する機会を得られる」</p>

<p>コードが光のブロックに吸い込まれた。</p>

<p>巨大なブロックが、小さなブロックの山に分割された。各ブロックは、順番にコールスタックに積まれては、処理されては、次のブロックへと入れ替わった。</p>

<p>「素晴らしい」ループが初めて表情を変えた。「しかし、これだけでは不十分だ」</p>

<p>ループがもう一度手をかざした。</p>

<p>今度は、マイクロタスクキューに、無数の光の玉が現れた。光の玉は、次々と新しい光の玉を生み出していた。</p>

<p>「マイクロタスクの無限生成」ループは言った。「あるマイクロタスクが、新たなマイクロタスクを生成する。その連鎖が止まらない」</p>

<p>「飢餓状態だ」カイトは顔をしかめた。「マイクロタスクが優先されるから、通常のタスクが永遠に実行されない」</p>

<p>「どうするの？」リアムが焦った。</p>

<p>「マイクロタスクの生成を制御する」カイトは答えた。「または、マイクロタスクを使わずに、通常のタスクとして処理する」</p>

<p>カイトは新たなコードを書いた。</p>

<pre><code>// 悪い例：マイクロタスクの無限生成
function infiniteMicrotasks() {
    Promise.resolve().then(() => {
        infiniteMicrotasks();
    });
}

// 良い例：タスクとして処理
function controlledTasks() {
    return new Promise(resolve => {
        setTimeout(() => {
            // 処理
            resolve();
        }, 0);
    });
}

// または、バッチ処理
function batchedMicrotasks(items, batchSize = 10) {
    let index = 0;
    
    function processBatch() {
        const end = Math.min(index + batchSize, items.length);
        while (index < end) {
            // 処理
            index++;
        }
        
        if (index < items.length) {
            return Promise.resolve().then(processBatch);
        }
    }
    
    return processBatch();
}</code></pre>

<p>コードが光の玉に向かって飛んだ。</p>

<p>無限に増殖していた光の玉が、徐々に減少し始めた。新しいコードが、無限ループを抑制していた。</p>

<p>「良い」ループが評価した。「しかし、まだ終わりではない」</p>

<p>ループが三度手をかざした。</p>

<p>今度は、イベントループの円環自体が揺らぎ始めた。</p>

<p>「イベントループの妨害」ループは言った。「外部からの干渉で、ループが不安定になっている」</p>

<p>「外部？」カイトは警戒した。「誰が？」</p>

<p>「古代の魔法使いたちが残した、最後の課題だ」ループは答えた。「イベントループを理解するだけでは不十分。イベントループを守る力も必要だ」</p>

<p>円環の向こう側から、黒い霧が漂ってきた。</p>

<p>霧の中から、複数の影が現れた。影は、かつてカイトたちが倒した守護者たちの形をしていた——同期の守護者、非同期の守護者、Promiseの守護者……。</p>

<p>「過去の試練が、ここで統合されている」カイトは気づいた。「これまでの全ての概念が、イベントループの中で交差する」</p>

<p>「どうする？」エリナがカイトを見た。</p>

<p>カイトは考えた。</p>

<p>「四で分担する」カイトは決断した。「イベントループには、複数の種類のタスクがある。それぞれの特性に応じて、分担して対処する」</p>

<p>「どう分担する？」リアムが尋ねた。</p>

<p>「エリナは、マイクロタスクキューを守ってくれ」カイトは指示した。「マイクロタスクは優先度が高い。素早い判断が必要だ」</p>

<p>「わかった」エリナが頷いた。</p>

<p>「リアムは、タスクキューを守ってくれ」カイトは続けた。「通常のタスクは量が多い。耐久力が必要だ」</p>

<p>「任せろ」リアムが剣を構えた。</p>

<p>「セシリアは、コールスタックを監視してくれ」カイトは言った。「実行中の処理が長すぎないか、常にチェックして」</p>

<p>「了解」セシリアが杖を構えた。</p>

<p>「俺は、イベントループそのものを守る」カイトは円環に向かって歩き出した。「ループが止まらないように、全力を尽くす」</p>

<p>「一人で大丈夫？」ループが初めて心配そうな表情を見せた。</p>

<p>「一人じゃない」カイトは仲間たちを振り返った。「俺たちには、チームワークがある」</p>

<p>戦いが始まった。</p>

<p>黒い霧から現れた影たちが、各領域に襲いかかった。</p>

<p>エリナは、マイクロタスクキューを守るために、光の矢を放った。矢は、影の攻撃を的確に阻止した。エリナの判断の速さが、マイクロタスクの性質と共鳴していた。</p>

<p>リアムは、タスクキューを守るために、剣で影を切り裂いた。大量の敵が押し寄せても、リアムは動じなかった。タスクキューの粘り強さと、リアムの忍耐力が呼応していた。</p>

<p>セシリアは、コールスタックを監視しながら、魔法でサポートした。処理が長くなりそうな時は、タイミングよく介入して、処理を分割した。セシリアの冷静さが、コールスタックの安定性と同調していた。</p>

<p>そして、カイトは、イベントループそのものを守るために、複雑な魔法を唱えた。</p>

<pre><code>// イベントループの安定化
function stabilizeEventLoop() {
    const originalPush = taskQueue.push;
    const originalUnshift = microtaskQueue.unshift;
    
    // タスクの監視
    taskQueue.push = function(task) {
        if (isMalicious(task)) {
            console.warn("悪意のあるタスクを検出、ブロックします");
            return;
        }
        return originalPush.call(this, task);
    };
    
    // マイクロタスクの監視
    microtaskQueue.unshift = function(task) {
        if (isMalicious(task)) {
            console.warn("悪意のあるマイクロタスクを検出、ブロックします");
            return;
        }
        return originalUnshift.call(this, task);
    };
    
    // ループの健全性チェック
    setInterval(() => {
        if (callStack.length > MAX_STACK_SIZE) {
            console.error("コールスタックが過負荷です");
            callStack.clear();
        }
        if (microtaskQueue.length > MAX_MICROTASKS) {
            console.warn("マイクロタスクが過多です、一部を削除します");
            microtaskQueue.splice(MAX_MICROTASKS);
        }
    }, 1000);
}</code></pre>

<p>魔法が、イベントループ全体を包み込んだ。</p>

<p>黒い霧が、魔法に触れて消えていった。影たちも、徐々に薄くなっていった。</p>

<p>「やった？」エリナが期待を持った。</p>

<p>「まだだ」カイトは油断しなかった。「中心に、何かいる」</p>

<p>黒い霧の中心から、最後の影が現れた。</p>

<p>それは、カイトの影だった。</p>

<p>「また俺か」カイトは苦笑した。</p>

<p>「イベントループは、君の思考パターンを学習した」影のカイトが言った。「君がどのようなコードを書くか、どのような対策をとるか、全て予測している」</p>

<p>「じゃあ、俺の戦い方は通じないってこと？」</p>

<p>「その通り」影のカイトが冷笑した。「君の知っている知識、君の使える技術——全て、私も知っている」</p>

<p>カイトは考えた。</p>

<p>「知識が同じなら、創造性で勝負するしかない」カイトは言った。</p>

<p>「創造性？」影のカイトが眉をひそめた。</p>

<p>「イベントループは、単なる仕組みじゃない」カイトは説明した。「そこには、選択の余地がある。どのタスクを優先するか、どの処理を分割するか、いつ待つか、いつ進むか——それらは全て、プログラマーの選択だ」</p>

<p>「だから？」</p>

<p>「俺は、仲間を選ぶ」カイトは言った。「一人で全てを背負うんじゃなくて、仲間と分担する。それが、俺のプログラミング哲学だ」</p>

<p>カイトは、仲間たちに合図を送った。</p>

<p>エリナ、リアム、セシリア——三人が、カイトの意図を理解した。</p>

<p>「共鳴！」カイトが叫んだ。</p>

<p>四人の力が、一つになった。</p>

<p>第34章で習得した共鳴の力が、イベントループの中で発動した。四人の処理が、同期的に結合した。</p>

<p>「これは……」影のカイトが戸惑った。「同期的な処理が、イベントループの中で？ そんなはずはない」</p>

<p>「イベントループは、並行性を管理する仕組みだ」カイトは言った。「でも、協調性を排除しているわけじゃない。四つの処理が、協力して一つの目的を達成する——それは、イベントループの概念と矛盾しない」</p>

<p>四人の光が、影のカイトを包み込んだ。</p>

<p>影のカイトが、ゆっくりと消えていった。</p>

<p>「君の選択は……正しいのかもしれない」影のカイトが最後に言った。「イベントループは、一人のためにあるんじゃない。全ての処理のためにあるんだ」</p>

<p>影が完全に消えた。</p>

<p>空間が静かになった。</p>

<p>「勝った」リアムが息を吐いた。</p>

<p>「本当に？」エリナが不安そうに尋ねた。</p>

<p>「本当に」ループが近づいてきた。「君たちは、イベントループの試練をクリアした」</p>

<p>ループの体が、徐々に実体化していった。半透明だった体が、人間と同じような質感になっていく。</p>

<p>「君たちの共鳴が、私を解放した」ループは言った。「私は、イベントループそのものだったから、自由意志を持てなかった。でも、君たちの協調の力が、私に『選択』という概念を与えた」</p>

<p>「選択？」セシリアが尋ねた。</p>

<p>「どのタスクを優先するか、どの処理を待たせるか——それは、私の判断だった」ループは説明した。「でも、判断には責任が伴う。私は、その責任から逃げられなかった」</p>

<p>「今は？」カイトが尋ねた。</p>

<p>「今は、君たちのおかげで、判断の意味を理解した」ループは微笑んだ。「イベントループは、単なる仕組みじゃない。そこには、選択と責任と、そして希望がある」</p>

<p>ループが手をかざした。</p>

<p>空間の奥に、新たな扉が現れた。</p>

<p>「次は、最後の試練だ」ループは言った。「『メモリ管理の深淵』——全ての終わりと始まりの場所」</p>

<p>「メモリ管理」カイトが呟いた。「ガベージコレクションか」</p>

<p>「そう」ループは頷いた。「不要になったものを手放す勇気、必要なものを守る決意——それらが問われる」</p>

<p>四人は、新たな扉の前に立った。</p>

<p>「いよいよ終わりが近い」カイトは言った。</p>

<p>「終わりというより、新しい始まりかも」エリナが言った。</p>

<p>「どちらにしても」リアムが剣を握り直した。「俺たちは進む」</p>

<p>「ええ」セシリアが微笑んだ。「一緒に」</p>

<p>四人は、メモリ管理への扉を開いた。</p>

<p>地下の迷宮は、さらに深く——そして、より本質的な場所へと続いていた。</p>

<p><strong>【プログラミング解説】</strong></p>

<ul>
<li><strong>イベントループ</strong>：非同期処理を管理する仕組み。イベントを待ち、実行し、次のイベントを待つというループを繰り返す。</li>
<li><strong>コールスタック</strong>：現在実行中の関数の呼び出し履歴。LIFO（後入れ先出し）の構造。</li>
<li><strong>タスクキュー</strong>：実行待ちのタスクが並ぶキュー。FIFO（先入れ先出し）の構造。</li>
<li><strong>マイクロタスクキュー</strong>：タスクキューより優先度の高いキュー。Promiseの解決などがここに入る。</li>
<li><strong>飢餓状態（Starvation）</strong>：マイクロタスクが無限に生成されると、通常のタスクが永遠に実行されない現象。</li>
<li><strong>タスクの分割</strong>：長い処理を短い処理に分けて、イベントループが他の処理を実行できるようにする技法。</li>
</ul>
