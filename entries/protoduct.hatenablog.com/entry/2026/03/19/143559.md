<h3>第27章：関数の刻印</h3>

<p>変数を失った世界で、カイトは立ち尽くしていた。</p>

<p>周囲の空気が重い。まるでプログラムが凍りついたような静止感。エリナが駆け寄ってくるのが、スローモーションのように見えた。</p>

<blockquote>「カイト！ 大丈夫！？」</blockquote>

<p>カイトは自分の手を見下ろした。指先が微かに震えている。先ほどの戦闘で、彼は気づいてしまったのだ──この世界の魔法システムにおける、根本的な欠陥を。</p>

<blockquote>「……変数が、ない」</blockquote>

<p>その言葉に、リアムが眉をひそめた。</p>

<blockquote>「何の話だ？」</blockquote>

<p>カイトは深呼吸をして、震える声を抑え込んだ。</p>

<blockquote>「この世界の魔法は、すべてが静的（スタティック）なんだ。値を保持できない。代入できない。つまり……状態を持てない」</blockquote>

<p>セシリアが首をかしげる。</p>

<blockquote>「つまり、どういうことですの？」</blockquote>

<p>カイトは地面に指を走らせた。泥の中に、文字のような模様が浮かび上がる。</p>

<pre><code>// 通常の魔法詠唱
cast(fire, target=enemy);

// でもこれができない
int power = 10;
power = power * 2; // 再代入が効かない！</code></pre>

<p>カイトは顔を上げた。</p>

<blockquote>「俺たちがこれまで使ってきた魔法は、すべてワンショットだ。一発撃って、終わり。連続した処理ができないし、状態を引き継げない。それが、この世界の限界なんだ」</blockquote>

<p>エリナの顔色が変わった。</p>

<blockquote>「じゃあ、私たちの魔法も……」</blockquote>

<blockquote>「そう。基本は全部、単発の呪文だけ。組み合わせるには、その場で新しく詠唱し直さなきゃならない」</blockquote>

<p>重い沈黙が降りた。</p>

<p>だが、カイトの瞳の中で、何かが燃え始めていた。プログラマーとしての本能だ。制約を見つけたとき、その制約を乗り越える方法を探し出す衝動。</p>

<blockquote>「でも……もしかしたら、突破口があるかもしれない」</blockquote>

<p>リアムが身を乗り出した。</p>

<blockquote>「言ってみろ」</blockquote>

<p>カイトは再び地面に指を走らせた。今度は、もっと複雑な模様を描いていく。</p>

<blockquote>「変数が使えないなら、関数を使えばいい」</blockquote>

<p>エリナが瞬きをした。</p>

<blockquote>「関数……？」</blockquote>

<p>カイトは頷いた。</p>

<blockquote>「そう。入力を受け取って、出力を返す処理の塊。状態を持たなくても、処理を定義することはできるはずだ」</blockquote>

<p>彼は立ち上がり、両手を空にかざした。</p>

<pre><code>// 関数の定義
define spell.blade(slash) {
    return sharpness * velocity;
}</code></pre>

<p>すると──。</p>

<p>カイトの掌の上に、淡い光の紋様が浮かび上がった。それは単なる魔法陣ではない。規則的なパターンを持つ、幾何学的な刻印だった。</p>

<blockquote>「これは……！」</blockquote>

<p>セシリアが息を呑んだ。</p>

<blockquote>「魔術師でも扱えない高等術式……いえ、そもそ形が違いますわ」</blockquote>

<p>カイトは刻印を見つめながら、静かに言った。</p>

<blockquote>「これは『関数の刻印』だ。一度定義すれば、何度でも呼び出せる。引数を変えれば、出力も変わる。変数を使わずに、処理を再利用できるんだ」</blockquote>

<p>リアムが剣の柄に手をかけたまま、警戒心を解かずに尋ねた。</p>

<blockquote>「それで、何ができる？」</blockquote>

<p>カイトは刻印を握りしめた。</p>

<blockquote>「魔法の『関数化』。今までは毎回ゼロから詠唱していたけど、これからは『関数を呼び出す』だけでいい。しかも……」</blockquote>

<p>彼は一歩前に出た。</p>

<blockquote>「関数の中に、別の関数を入れ込むこともできる。『入れ子構造』だ」</blockquote>

<p>エリナが目を輝かせた。</p>

<blockquote>「つまり、魔法の組み合わせがもっと自由になるってこと？」</blockquote>

<blockquote>「その通り。変数なしで、再利用可能な魔法ライブラリを作れるんだ」</blockquote>

<p>そのときだった。</p>

<p>空気が、再び振動した。</p>

<p>先ほどの戦闘で現れた黒い影──「変数喰らい」とでも呼ぶべき存在──が、森の奥から姿を現した。今度は一体ではない。三体、四体……いや、十体以上が、音もなく近づいてくる。</p>

<blockquote>「来るぞ！」</blockquote>

<p>リアムが剣を抜いた。セシリアも杖を構える。</p>

<p>だが、カイトは動かなかった。彼はただ、掌の刻印を見つめていた。</p>

<blockquote>「カイト！ 何をしてるの！？」</blockquote>

<p>エリナの叫びにも、彼は応えない。</p>

<p>黒い影たちが、一斉に飛びかかってきた。</p>

<p>その瞬間──。</p>

<blockquote>「invoke spell.blade(slash)」</blockquote>

<p>カイトの口から、静かな言葉が漏れた。</p>

<p>次の瞬間、彼の手から光の刃が走った。それは単なる魔法攻撃ではない。複数の敵を同時に切り裂く、プログラム的な処理の連鎖だった。</p>

<pre><code>// 関数の連鎖呼び出し
invoke spell.blade(target[0]);
invoke spell.blade(target[1]);
invoke spell.blade(target[2]);
// ...ループで処理</code></pre>

<p>影たちは一瞬で消滅した。いや、正確には「処理された」のだ。</p>

<p>エリナが呆然と立ち尽くした。</p>

<blockquote>「今の……何？」</blockquote>

<p>カイトは掌の刻印を消した。</p>

<blockquote>「関数呼び出しの応用だ。一度定義した魔法を、複数の対象に適用しただけ」</blockquote>

<p>リアムが剣を納めた。</p>

<blockquote>「……驚いたな。本当に、魔法を『プログラミング』してやがったのか」</blockquote>

<p>カイトは小さく笑った。</p>

<blockquote>「まだ始まったばかりだよ。関数が使えるなら……クラスだって定義できるかもしれない」</blockquote>

<p>セシリアが首をかしげた。</p>

<blockquote>「クラス……ですの？」</blockquote>

<p>カイトは空を見上げた。雲の合間から、夕日が差し込んでいる。</p>

<blockquote>「データと処理をまとめた『型』のことだ。もしクラスが作れれば……俺はこの世界で、新しい魔法体系そのものを作れるかもしれない」</blockquote>

<p>エリナがカイトの隣に立った。</p>

<blockquote>「カイト、あなたって本当に……」</blockquote>

<p>彼女は言葉を探して、それから小さく笑った。</p>

<blockquote>「変わってるわね」</blockquote>

<p>カイトも笑った。</p>

<blockquote>「プログラマーだからな」</blockquote>

<p>夕暮れの森の中、四人の影が長く伸びていた。</p>

<p>変数のない世界で、カイトは関数を見つけた。そしてその先には、まだ見ぬ「クラス」と「オブジェクト」の概念が待っている。</p>

<p>彼の冒険は、まだ終わらない。</p>

<hr>

<p><strong>【用語解説】</strong></p>

<ul>
<li><strong>関数の刻印（Function Sigil）</strong>：魔法を関数として定義し、再利用可能にする術式。一度刻印すれば、引数を変えるだけで異なる効果を発揮できる。</li>
<li><strong>変数喰らい（Variable Eater）</strong>：この世界の魔法から「変数」を奪う存在。正体不明だが、古代の魔法実験の失敗で生まれたという説がある。</li>
<li><strong>静的魔法（Static Magic）</strong>：状態を持たない魔法。この世界の標準的な魔法体系。カイトが発見した「関数」は、これを動的に近づける第一步。</li>
</ul>
