第31章：try-catch文の守護

地下への階段は、予想以上に深く続いていた。カイトたちは暗闇の中を慎重に進んでいた。壁には薄っすらと光る苔が生えており、それが唯一の光源だった。

「長い階段ね」

エリナがため息をついた。

「どれくらい下っているのかしら」
「少なくとも百段は超えている」

リアムが数えていた。

「戦場で学んだことだが、階段の段数を数える癖がついていてな」

セシリアが魔法で周囲を探査した。

「地下空間が広がっているわ。この階段の先に、何か大きな構造がある」

カイトは頷いた。彼の programmer's intuition が、何か重要なものが待っていると告げていた。

「例外処理の試練……予期せぬエラーに対処する方法」

カイトは独り言のように言った。

「プログラミングにおいて、エラーは避けられないものだ。ファイルが見つからない、ネットワークが切断される、メモリが不足する……どんなに完璧なコードを書いても、外部要因でエラーは起こる」
「それをどうやって対処するの？」

エリナが尋ねた。

「try-catch文を使う。試してみて、エラーが起きたら捕獲する。それが基本概念だ」

カイトは空中にコードを描いた。


try {
    // 試みる処理
    openAncientDoor();
    enterSecretChamber();
} catch (error) {
    // エラーが起きた時の処理
    console.log("エラーが発生しました: " + error.message);
    activateBackupPlan();
}
「tryブロックの中に、やりたい処理を書く。もしエラーが起きたら、catchブロックに処理が移る。こうすることで、プログラム全体がクラッシュするのを防げる」

リアムが興味深そうに言った。

「戦場で言えば、作戦が失敗した時の撤退計画のようなものか」
「まさに。tryが作戦実行、catchが撤退計画だ」

その時、階段が終わった。目の前に、巨大な地下空間が広がっていた。

そこは、古代の研究所のようだった。無数の机があり、その上には奇妙な装置が置かれている。壁には、複雑な魔法陣が描かれていた。そして、部屋の中央には、一本の石柱が立っていた。

「ここは……」

セシリアが目を見開いた。

「古代の魔法実験場だわ。かつて、偉大な魔法使いたちがここで研究を行っていた」
「何を研究していたんだ？」

カイトが尋ねた。

「不明よ。ただ、一つだけ言えることがある」

セシリアは慎重に言葉を選んだ。

「古代の実験は、しばしば暴走した。その結果、封印されることになったものも多い」

カイトは嫌な予感を感じた。

「つまり、ここにはエラーの原因になるものが大量にあるってことか」
「その通り」

四人は慎重に部屋の中を進んだ。カイトは、各机の上にある装置を観察した。

  • 輝く水晶球——中で何かが渦巻いている
  • 複雑な歯車の仕掛け——一部が錆びている
  • 古代の書物——ページが散乱している
  • 魔力を帯びた宝石——不安定に明滅している
「どれもこれも、エラーの温床だ」

カイトは頭を抱えた。

「触れた瞬間に暴走するかもしれない」
「じゃあ、触れなきゃいいじゃない」

エリナが言った。

「そうもいかない。見ろ」

カイトは石柱を指差した。石柱には、古代文字が刻まれていた。


TRY {
    ACTIVATE_ALL_DEVICES();
} CATCH {
    UNLOCK_NEXT_PATH();
}
「すべての装置を起動しろ、だと？」

リアムが眉をひそめた。

「罠じゃないのか？」
「いや、試練だ。try-catchの実践」

カイトは理解した。

「装置を起動する時、エラーが起きる可能性がある。でも、エラーをcatchして適切に対処すれば、先に進める」

セシリアが分析した。

「つまり、私たちは同時に二つのことをしなければならないわ。装置を起動することと、エラーに対処すること」
「その通り。役割分担しよう」

カイトは作戦を立てた。

「エリナとリアムは、物理的な装置の起動を担当して。セシリアと私は、魔法のエラーをcatchする」

四人は頷き合った。

「始めよう」

カイトは、まず水晶球に近づいた。


try {
    let crystal = document.getElementById("glowing-crystal");
    crystal.activate();
    
    // 水晶の中身を確認
    let contents = crystal.getContents();
    console.log("水晶の中身: " + contents);
    
} catch (error) {
    if (error.type === "UNSTABLE_MANA") {
        // 不安定な魔力を鎮める
        SPELL {
            type: "MANA_STABILIZER"
            target: crystal
            effect: "CALM"
        }
    } else if (error.type === "SEALED_ENTITY") {
        // 封印された存在が解放されそうになった
        SPELL {
            type: "RESEAL"
            target: crystal
            power: "MAXIMUM"
        }
    } else {
        // 未知のエラー
        console.log("未知のエラー: " + error.message);
        alert("カイト、何か異常が起きた！");
    }
}

カイトは水晶球に手をかざした。

「起動！」

水晶球が激しく明滅した。中の渦が高速で回り始める。そして——

BOOM!

水晶球が爆発した！ しかし、カイトの魔法が即座に反応した。


ERROR DETECTED: UNSTABLE_MANA
ACTIVATING CATCH BLOCK...
EXECUTING: MANA_STABILIZER

カイトの手から鎮静化の魔法が放たれ、爆発の衝撃を中和した。水晶球は静かに砕け散った。

「一つクリア」

カイトは息を吐いた。

「次」

リアムが歯車の仕掛けに近づいた。

「俺は機械には詳しくないが……回せばいいんだな？」

リアムは力任せに歯車を回した。

GIRI GIRI GIRI——

錆びた歯車が悲鳴を上げた。そして、装置全体が揺れ始めた。

「エラー発生！」

カイトが叫んだ。


try {
    gearMechanism.rotate();
} catch (error) {
    if (error.type === "RUST_DAMAGE") {
        // 錆びた部分を修復
        SPELL {
            type: "RESTORATION"
            target: gearMechanism
            effect: "REMOVE_RUST"
        }
        // 再試行
        gearMechanism.rotate();
    } else {
        throw error; // 処理できないエラーは再スロー
    }
}

カイトの修復魔法が歯車に照射された。錆が剥がれ落ち、歯車が滑らかに回り始めた。

「リアム、続けて！」

リアムは再び歯車を回した。今度はスムーズに回転した。装置の奥から、カチッという音がした。

「二つクリア」

次は、古代の書物だった。エリナがページを拾い集めた。

「文字が読めないわ。古代語ね」

セシリアが近づいた。

「私が読んでみる」

セシリアはページを慎重にめくった。その瞬間、ページから黒い霧が立ち上った！

「罠！」

エリナが叫んだ。黒い霧がセシリアを包み込もうとした。

しかし、カイトのcatchブロックが発動した。


try {
    let book = ancientTome;
    book.open();
    book.read();

    if (error.type === "CURSE_ACTIVATED") {
        // 呪いを無効化

カイトの手から聖なる光が放たれた。黒い霧が光に触れ、浄化されて消滅した。セシリアは無事だった。

「ありがとう、カイト」
「catchのおかげだ」

最後に残ったのは、魔力を帯びた宝石だった。不安定に明滅している。

「これが一番危険そうだ」

カイトは慎重に宝石に近づいた。

「みんな、下がっていてくれ」

カイトは、最強のtry-catch構文を構築した。


class UltimateTryCatch {
    constructor() {
        this.errorLog = [];
        this.recoveryAttempts = 0;
        this.maxRecoveryAttempts = 3;
    }
    
    async execute(target) {
        try {
            // 宝石の起動を試みる
            let gem = await this.activateGem(target);
            return gem;
            
        } catch (error) {
            // エラーログに記録
            this.errorLog.push({
                timestamp: Date.now(),
                type: error.type,
                message: error.message,
                severity: error.severity
            });
            
            // エラーの種類に応じて対処
            switch (error.type) {
                case "MANA_OVERLOAD":
                    return this.handleManaOverload(target);
                    
                case "DIMENSIONAL_RIFT":
                    return this.handleDimensionalRift(target);
                    
                case "ANCIENT_SEAL_BREAKING":
                    return this.handleSealBreaking(target);
                    
                default:
                    // 未知のエラー
                    if (this.recoveryAttempts < this.maxRecoveryAttempts) {
                        this.recoveryAttempts++;
                        return this.execute(target); // 再試行
                    } else {
                        throw new Error("Recovery failed after " + this.maxRecoveryAttempts + " attempts");
                    }
            }
        }
    }
    
    handleManaOverload(gem) {
        SPELL {
            type: "MANA_DRAIN"
            target: gem
            rate: "CONTROLLED"
        }
    }
    
    handleDimensionalRift(gem) {
        SPELL {
            type: "DIMENSION_LOCK"
            target: gem
            duration: "PERMANENT"
        }
    }
    
    handleSealBreaking(gem) {
        SPELL {
            type: "EMERGENCY_RESEAL"
            target: gem
            power: "ULTIMATE"
        }
    }
}

カイトは深呼吸をした。

カイトが宝石に触れた瞬間、強烈な光が放たれた！


ERROR DETECTED: MANA_OVERLOAD

カイトの魔法が過剰な魔力を吸収した。しかし——


ANOTHER ERROR DETECTED: DIMENSIONAL_RIFT
EXECUTING CATCH BLOCK...
ACTIVATING: DIMENSION_LOCK

空間が歪み始めた！ 異次元への扉が開こうとしていた。カイトは即座に次元を固定した。


ANOTHER ERROR DETECTED: ANCIENT_SEAL_BREAKING
EXECUTING CATCH BLOCK...
ACTIVATING: EMERGENCY_RESEAL

宝石の中から、何かが解放されようとしていた。カイトは全力で封印し直した。

三つのエラーが同時に発生し、三つのcatchブロックが連鎖的に発動した。カイトの魔力が激しく消耗していく。

「カイト！」

エリナが叫んだ。

「大丈夫！ まだ……いける！」

カイトは歯を食いしばった。try-catchの本質は、単にエラーを捕まえることではない。エラーに対して、適切かつ迅速に対処することだ。


ALL ERRORS HANDLED SUCCESSFULLY
GEM ACTIVATION COMPLETE

宝石が静かに輝き始めた。安定した、穏やかな光だった。

カイトは膝をついた。疲労が全身を襲った。

「やった……のか？」

リアムがカイトを支えた。

「ああ。見ろ」

石柱が光り始めた。そして、地下空間の奥にある壁が開いた。新たな通路が現れた。

「try-catchの試練、クリアだ」

セシリアが安堵の息を吐いた。

「カイト、すごいわ。三つのエラーを同時に処理するなんて」
「catchブロックの連鎖だ」

カイトは立ち上がった。

「一つのtryに対して、複数のcatchを用意できる。エラーの種類ごとに、最適な対処法を実行する。それが、try-catchの真髄だ」

エリナがカイトの肩に手を置いた。

「でも、無理はしないでね」
「ありがとう。でも、まだ終わっていない」

カイトは新たな通路を見つめた。

「try-catchには、もう一つ重要な概念がある。finallyだ」
「finally？」

リアムが首を傾げた。

「tryブロックが成功しても失敗しても、最後に必ず実行される処理だ。後始末と言ってもいい」

カイトは空中にコードを描いた。


try {
    // 試みる処理
    enterNextTrial();
} catch (error) {
    // エラー処理
    handleCrisis(error);
} finally {
    // 成功・失敗に関わらず実行
    restoreMana();
    healWounds();
    prepareForNextChallenge();
}
「finallyは、結果にかかわらず実行される。だから、リソースの解放や、後始末に使われる」

カイトは仲間たちを見回した。

「僕たちの冒険も同じだ。成功しても失敗しても、最後には必ず何かを学ぶ。それが、finallyの精神だ」

四人は新たな通路へと足を踏み入れた。

通路の先には、さらに深い闇が広がっていた。しかし、カイトの心には光があった。

try-catch-finally。エラーを恐れず、対処し、学ぶ。それが、プログラマーとしての生き方だった。

「行こう、みんな。次の試練が待っている」

カイトの言葉に、三人は頷いた。彼らは、闇の中へと消えていった。

finallyの先に、何が待っているのか。それは、まだ誰も知らない。
