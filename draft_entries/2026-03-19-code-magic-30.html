第30章：break文の解放

無限ループの牢獄から脱出して三日が経っていた。カイトたちは古代図書館の回廊を進みながら、前回の教訓を反芻していた。while(true)の恐怖。終了条件なき繰り返しの悪夢。それは、カイトにとって決して忘れられない体験となっていた。

「それにしても、脱出できてよかったわ」

エリナがため息交じりに言った。

「正直、あそこで終わるかと思った」
「break文が見つからなかったら、終わっていたな」

カイトは冷静に答えた。しかし、その声には安堵の色が混じっていた。

無限ループの牢獄。それは、古代の魔法使いが作った試練の場だった。空間自体がループ構造になっており、一度入ると特定の条件を満たすまで出られない。カイトたちは三日間、同じ廊下を歩き続けた。何度同じ風景を見たかわからない。何度同じ敵と戦ったかわからない。

ループの中では、記憶すら曖昧になった。昨日のことが今日のことのように感じる。今日のことが明日のことのように感じる。時間の概念が溶解し、残るのは永遠の繰り返しだけ。

「でも、おかげでbreak文の重要性が身に染みた」

カイトは空中に指で文字を描いた。


LOOP_PRISON {
    while (true) {
        // 繰り返される現実
        walkCorridor();
        fightEnemies();
        solvePuzzles();
        
        // 脱出条件を見つけるまで続く
        if (foundExitCondition()) {
            break; // これが救いだった
        }
    }
}
「break。たった一語のコードだけど、これがあるかないかで、ループが救いか地獄か分かれる」

リアムが頷いた。

「戦場でも同じだ。撤退のタイミングを見極められない指揮官は、軍を壊滅させる」
「まさに。ループは強力だが、breakなしには破滅を招く」

セシリアが興味深そうに尋ねた。

「他にもループを終わらせる方法はあるの？」

カイトは少し考え、答えた。

「return文もある。関数全体を終了して、呼び出し元に戻る。ただ、これはbreakとは少し意味が違う」
「どう違うの？」
「breakはループを抜けるだけ。returnは、今いる処理全体を終了させる。例えば、ある部屋から出るのがbreakだとしたら、returnは建物全体から出るようなものだ」

エリナが眉をひそめた。

「複雑ね」
「プログラミングはそういうものだ。一つのことを達成するのに、複数の方法がある。どれを選ぶかで、結果が変わる」

四人は回廊の突き当たりに到着した。そこには、巨大な扉があった。扉には複雑な幾何学模様が刻まれており、中央には「EXIT」という文字が浮かんでいた。

「出口……？」

エリナが期待を込めて言った。

「いや、そう簡単にはいかないだろう」

カイトは警戒した。古代図書館にこんな分かりやすい出口があるはずがない。

予想通り、扉は開かなかった。その代わり、扉から声が聞こえた。

「breakを学びし者よ。次の試練は、continueだ」

重々しい声だった。カイトは眉をひそめた。

「continue？ breakじゃないのか？」

扉の声が答えた。

「breakはループを終わらせる。continueは、今の回をスキップして次に進む。似ているようで、全く異なる概念だ」

カイトは理解した。


for (let i = 0; i < enemies.length; i++) {
    if (enemies[i].isAlive === false) {
        continue; // 死んでいる敵はスキップ
    }
    
    attack(enemies[i]); // 生きている敵だけ攻撃
}
「なるほど。特定の条件で処理をスキップし、次のループに進む。無駄な処理を省くのに役立つ」

扉がゆっくりと開いた。中から、眩しい光が溢れ出した。

「試練の間へようこそ」

四人は光の中へと足を踏み入れた。

光が収まると、そこは広大な闘技場だった。観客席には誰もいない。中央には、円形のステージがあった。そして、ステージの上には無数の魔石が浮かんでいた。

「これは……？」

リアムが剣に手をかけた。

「魔石の群れか？」
「違う。見てみろ」

カイトは魔石を観察した。それぞれの魔石には、異なる色とシンボルがあった。そして、一部の魔石はひび割れていた。

「壊れた魔石と、無傷の魔石。二種類がある」

セシリアが分析した。

「壊れた魔石は魔力を失っている。無傷の魔石は、まだ魔力を保持しているわ」

カイトは頷いた。

「試練の内容は、無傷の魔石だけを破壊することだ。continueを使って、壊れた魔石をスキップしながら」

エリナが不思議そうに尋ねた。

「なんで壊れた魔石をスキップする必要があるの？ 全部壊せばいいじゃない」
「壊れた魔石を攻撃すると、負の魔力が放出される。周囲に被害を与えるんだ。だから、無傷の魔石だけを的確に狙う必要がある」

カイトは魔法の構築を開始した。


class SelectiveDestruction {
    constructor() {
        this.targetQueue = [];
    }
    
    scan(stones) {
        for (let stone of stones) {
            if (stone.isBroken) {
                continue; // 壊れた魔石はスキップ
            }
            
            this.targetQueue.push({
                id: stone.id,
                position: stone.position,
                defense: stone.defense
            });
        }
    }
    
    execute() {
        for (let target of this.targetQueue) {
            this.destroy(target);
        }
    }
    
    destroy(target) {
        SPELL {
            type: "PIERCING_LANCE"
            power: target.defense * 1.2
            precision: 100
            target: target.position
        }
    }
}
「準備完了だ」

カイトは目を閉じた。魔導ネットワークに接続し、魔石の状態をスキャンする。

「三十七個の無傷魔石を破壊する。continueで壊れた六十三個をスキップ」

カイトは右手を掲げた。

「選択破壊魔法、シーク・アンド・デストロイ！」

カイトの手から、三十七本の光の槍が放たれた。それぞれの槍は、的確に無傷の魔石だけを狙っていた。壊れた魔石の近くを通っても、一切触れない。

光の槍が魔石に突き刺さるたび、澄んだ音が響いた。パリーン、パリーン、パリーン。続けざまに三十七個の魔石が砕け散った。

一方で、壊れた魔石は一つも触れられていない。カイトの魔法は、完璧な精度で無傷の魔石だけを狙い撃ちした。

「すごい……」

エリナが感嘆の声を上げた。

「六十三個の邪魔な石を避けて、三十七個だけを正確に破壊するなんて」
「continueの力だ」

カイトは肩で息をした。

「条件に合わないものをスキップし、必要なものだけを処理する。これが、continueの実践的応用だ」

リアムが腕を組んだ。

「戦場でも使えそうだな。民間人がいる場合、敵だけを狙って攻撃できる」
「その通り。ただ、continueにも限界がある」

カイトは冷静に警告した。

「continueは、あくまでスキップするだけ。スキップしたものに対しては、何のアクションも取れない。もし、スキップしたものが後で脅威になったらどうする？」

セシリアが首を傾げた。

「つまり、壊れた魔石が後で復活するかもしれないってこと？」
「あり得る話だ。その場合、continueではなく、別の対処が必要になる」

カイトの言葉通り、その瞬間、壊れた魔石が震え始めた。六十三個の壊れた魔石が、一斉に赤い光を放った。

「予想的中！」

カイトは叫んだ。

「壊れた魔石が再生を始めた！」

壊れた魔石の破片が集まり、形を成し始めた。一つ、また一つ。六十三個の魔石が、徐々に再生していく。

「continueだけじゃ対処できない！」

エリナが焦った。

「どうするの、カイト！？」

カイトは考えた。continueでスキップしたものが、後で脅威になる。この状況をどう打破するか？

「breakでループを抜けるわけにはいかない。まだ戦いは終わっていない」

カイトは唇を噛んだ。

「continueもbreakも、今の状況には適さない。必要なのは……」

その時、カイトの脳裏にアイデアが閃いた。

「continueもbreakも、ループの中での制御文だ。でも、ループ自体を変えられたらどうなる？」

カイトは、新しい魔法を構築し始めた。


class LoopTransformer {
    transform(originalLoop) {
        // 単純なfor文から、条件付きループへ変換
        return {
            type: "CONDITIONAL_WHILE",
            condition: (stone) => !stone.isRegenerating,
            action: (stone) => this.preventRegeneration(stone),
            fallback: (stone) => this.destroy(stone)
        };
    }
    
    preventRegeneration(stone) {
        SPELL {
            type: "MANA_SEAL"
            target: stone.position
            effect: "PREVENT_REGENERATION"
            duration: "INFINITE"
        }
    }
    
    destroy(stone) {
        SPELL {
            type: "ANNIHILATION"
            target: stone.position
            power: "MAXIMUM"
        }
    }
}
「ループの変換だ！ 単純なスキップではなく、条件に応じてアクションを変える！」

カイトは両手を広げた。

「ループ変換魔法、トランスフォーム・ループ！」

カイトから波紋のような魔力が広がった。それは、闘技場全体を包み込んだ。

魔力の波紋が壊れた魔石に触れた瞬間、魔石の再生が止まった。封印されたのだ。

「今だ！ 全面的な破壊に切り替える！」

カイトは魔法を展開した。今度は、continueでスキップせず、すべての魔石を破壊する。

「全体破壊魔法、グランド・フィナーレ！」

巨大な光の奔流が、闘技場を飲み込んだ。すべての魔石が、一瞬で消滅した。

静寂が戻った。ステージには、何も残っていなかった。

「勝った……」

リアムが息を吐いた。

「continueの限界を乗り越えたな」
「ああ。continueは便利だが、状況が変われば通用しない。柔軟な対応が必要だ」

カイトは疲労を感じながらも、満足感を味わっていた。新しいプログラミングの概念を、実戦で習得できた。

その時、闘技場の中央に階段が現れた。地下へと続く階段だった。

「次の試練は、地下か」

セシリアが言った。

「何が待っているのかしら」

カイトは階段の先を見つめた。暗くて、底が見えない。

「breakもcontinueも学んだ。次は……例外処理かもしれない」
「例外処理？」

エリナが首を傾げた。

「予期せぬエラーに対処する方法だ。try-catch文とも呼ばれる」

カイトは階段に足をかけた。

「どんなに完璧なプログラムでも、エラーは起きる。その時、どう対処するか。それが、例外処理の本質だ」

四人は、地下への階段を下りていった。闇が彼らを包み込む。しかし、カイトの目には、希望の光が宿っていた。

プログラミングの概念を一つ一つ習得し、魔法の使い手として成長していく。異世界での冒険は、まだ始まったばかりだった。

「行こう、みんな。次の試練が待っている」

カイトの言葉に、三人は頷いた。彼らは、闇の中へと消えていった。

地下の奥底で、何が待っているのか。それは、まだ誰も知らない。
